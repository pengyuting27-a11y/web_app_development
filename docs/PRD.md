# PRD：線上算命系統

## 1. 專案概述
- **背景與動機**：許多人在生活中遇到選擇困難、面臨迷航或單純對未來好奇時，會尋求算命與占卜的指引。本系統旨在幫助使用者打破實體的限制，讓使用者能在家透過網路，隨時隨地進行線上算命、占卜與抽籤，獲得心靈上的寄託與建議。
- **目標用戶**：對命理、星座、抽籤有興趣的大眾，或是在生活、學業、工作中遇到瓶頸，尋求心理安慰與方向指引的數位原生代使用者。
- **核心價值主張**：提供靈活、準確分析與充滿趣味性的線上算命服務，並妥善保存個人的占卜紀錄，讓信仰與心靈指引變得觸手可及。

## 2. 功能需求
1. **線上抽籤與占卜**
   - 作為 **使用者**，我希望 **可以進行線上抽籤或占卜**，以便 **在需要時取得神明的指引或答案**。
2. **展示算命結果與解析**
   - 作為 **使用者**，我希望 **在算出結果後，能看到清晰詳細的籤詩解說或占卜分析**，以便 **理解這次算命的深層涵義與建議**。
3. **儲存算命結果**
   - 作為 **使用者**，我希望 **系統能記錄我過去的各項算命結果與日期**，以便 **我可以在日後回顧與印證當時的指引**。
4. **會員註冊與帳號管理**
   - 作為 **開發者/使用者**，我希望 **能有登入及註冊的功能**，以便 **專屬的儲存及保障我個人的測算歷史紀錄**。
5. **分享結果**
   - 作為 **使用者**，我希望 **能一鍵將我的好運算命結果分享或複製連結**，以便 **與親朋好友分享我的好運勢**。

## 3. 非功能需求
- **技術限制**：使用 Flask 框架 + Jinja2 模板引擎進行全端網頁開發，並使用 SQLite 作為輕量級關聯式資料庫。
- **效能與安全考量**：
  - 使用者密碼需經過雜湊化（Hash）加密儲存。
  - 各項算命的隨機數生成需確保充分的不可預測性。
  - 介面回饋需要迅速，抽籤動畫及結果顯示的延遲需低於 2 秒鐘。
  - 防止 SQL Injection 與 XSS 攻擊。

## 4. MVP 範圍
- **Must Have (必須要有)**：線上抽籤功能、算命結果顯示、會員系統（註冊/登入）、儲存與查看算命歷史紀錄。
- **Should Have (應該要有)**：結果分享功能（如單純的內容複製）。
- **Nice to Have (可以要有)**：精美的神明抽籤介面動畫、更多元的算命類型（塔羅牌、生辰八字）、進階的付費解厄功能。
